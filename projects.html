Thought for food

This is my current capstone project. Which to me just means the project that took the longest to complete.

The project itself is a basic meal planning application, the idea is that you are able to search for a meal and the application will give information on total carbs, fat and protein. Then if its necassary for you it will generate a list of ingredients for all selected meals.

Suggested revisions for this website include recipe information and improved styling. These concerns will be addressed in the thought for food version 2. Along with an aggregated ingredient list and a user system to save meals and recipes for reference will all be implemented when I have a lull in projects to complete

Untitled Go App (working title: go-app-v1)

This is my most ambitious project yet, and will be, upon completion, a project that showcases my abilities as a full stack web developer. The backend is written in go, and still has a little way to go. Whereas I am waiting to have a backend that can store all of the necessary info before I write any of the front end.

The skills that I have used so far are GoLang and MySQL, proposed technologies for the end are ReactJS, HTML, and CSS. I also plan to integrate Leaflet maps with an OpenMaps tileset, and will be using axios as my library for API calls. Although slower than Go, I plan to do any password hashing on the client side using JS which is to avoid any sensitive data being intercepted on public networks, and client side is where I am likely to sanitize inputs from users.

I believe this project will take approximately 3 months until completion, and I will keep this page updated with any changes to technologies or timeline. My plan is to deploy this through AWS when complete.
